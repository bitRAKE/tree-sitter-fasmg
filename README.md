# tree-sitter-fasmg

Tree-sitter grammar for [fasmg](https://flatassembler.net/), the flat
assembler g.

## Naming

This repo follows tree-sitter community convention:

- **`tree-sitter-{lang}`** — the **grammar** (this repo). Matches
  `tree-sitter-javascript`, `tree-sitter-python`, the npm package, and
  the Cargo crate name.
- **`{lang}-tree-sitter`** — reserved for *bindings* that wrap a
  grammar for a specific host runtime (editor plugin, language
  toolchain glue). Not currently used for fasmg.

If you find yourself reaching for the mirror form, you probably want a
new repo, not this one.

## Layout

- `grammar.js` — grammar source (edit this, then regenerate)
- `src/` — generated parser (`parser.c`, `scanner.c`, `grammar.json`,
  `node-types.json`); checked in as the source of truth
- `queries/` — `highlights.scm`, `locals.scm`, `folds.scm`
- `spec/` — JSON snapshots of keyword groups and tokenisation rules
- `test/corpus/` — tree-sitter `.txt` corpus tests
- `test/fixtures/` — pass/fail `.l4gs.g` fixtures (one file per
  syntactic feature; verified against fasmg `g.l4gs`)
- `bindings/rust/` — Rust binding crate (`tree-sitter-fasmg`)
- `baseline/` — corpus parse statistics over external fasmg corpora
  (defaults to Grysztar's `fasmg/packages` + `fasm2/include`);
  regenerated by `scripts/corpus-baseline.sh`
- `GRAMMAR_SURFACE.md` — node catalog + field contract for downstream
  consumers (LSPs, editors, analyzers — read this first)

## Tree-sitter CLI

The CLI is **not vendored**. The shim scripts resolve it from
`$TREE_SITTER_PATH` (defaults to `C:\third_party\tree-sitter`):

```bash
# Test the grammar against test/corpus/
bash scripts/ts.sh test

# Regenerate src/parser.c after grammar.js edits
bash scripts/ts.sh generate --js-runtime native
```

PowerShell:

```powershell
.\scripts\ts.ps1 test
.\scripts\ts.ps1 generate --js-runtime native
```

## Corpus baseline

`scripts/corpus-baseline.sh` parses every `.inc`/`.asm`/`.alm` file
under each configured corpus root, records the innermost ERROR span
for each failure, and writes `baseline/{results.csv,failures.tsv,
summary.md,raw-summary.json}`. Re-running into the same dir overwrites
the committed reference; passing an alternate dir prints a per-file
delta against it.

```bash
# Refresh the committed baseline.
bash scripts/corpus-baseline.sh

# Compare a candidate run against baseline/ without overwriting.
bash scripts/corpus-baseline.sh out/
```

Override the corpora set via `FASMG_CORPORA` (colon-separated MSYS
paths). Each root is searched recursively; report paths are tagged
with the basename of the root's parent directory.

```bash
export FASMG_CORPORA="/c/path/one:/c/path/two"
bash scripts/corpus-baseline.sh
```

Throughput (bytes/ms) and total parse time land in `summary.md`'s
totals table — the script is a regression net for both *which* files
parse and *how fast* they do.

To use a different install, point the env var at the directory holding
the binary:

```bash
export TREE_SITTER_PATH=/path/to/tree-sitter-cli-dir
```

The shim expects `tree-sitter.exe` (Windows) or `tree-sitter` (POSIX)
inside that directory. Pin the version that matches the committed
`src/parser.c`; see `package.json` devDeps for the version the current
parser.c was generated with.

## Bindings

- **Rust** (`bindings/rust/`) — exposed by the `tree-sitter-fasmg`
  crate defined in `Cargo.toml`. Consume as a path or git dep.

Other bindings (Node, Python, Swift, Go) are not currently set up. To
scaffold one:

```bash
bash scripts/ts.sh init bindings <name>
```
